## 🎲 场景生成规则

**每次生成 prompt 时，从以下模块中随机组合：**

```
[场景 1个] + [主题/角色 1个] + [景别 1个] + [视角 1个] + [设备 1个] + [裸露/服装 1-2个] = 场景设定
```

**随机性规则：**

- **私密场景**：优先手机/网络摄像头 + POV/偷窥视角 + 近景/特写 + 裸露程度高
- **正式场景**：优先单反/胶片 + 平视/仰视 + 中景/全景 + 裸露程度中低
- **偷情场景**：优先监控/针孔 + 偷窥视角 + 远景/全景（被抓拍感）+ 裸露程度中
- **风俗场景**：优先手机/胶片 + 偷窥/POV视角 + 中景/近景 + 裸露程度高
- **AV封面场景**：优先专业单反 + 平视 + 中景/全景 + 裸露程度中高、化妆精致
- **复古场景**：优先胶片相机 + 平视 + 中景/近景 + 裸露程度随机

**⚠️ 注意：**

- 场景设定放在 prompt 最前面，引导整体画面氛围
- 景别和视角要匹配（特写不适合远景，鸟瞰适合全景）
- 设备选择要和场景氛围匹配（卧室偷拍不适合专业单反，AV封面不适合手机自拍）
- 裸露/服装紧跟拍摄设备之后，包含裸露程度和具体衣物状态
- 裸露程度要和场景/主题匹配（温泉→全裸，OL→制服半解，AV封面→裸露且有封面感）
- 每个 prompt 只能选 1 个场景 + 1 个主题 + 1 个景别 + 1 个视角 + 1 个设备 + 1-2 个裸露/服装

### 🚗 场景合理性铁律（写场景前必须过！）

**核心：先想"现实里做得到吗"，再写提示词。**

**三步检查：**
1. **物理可行性**：有支撑点吗？重心在哪？会摔倒吗？
2. **动作矛盾**：一个人能同时做两个需要专注的动作吗？
3. **空间关系**：两人的器官能接触到吗？

**❌ 禁止组合（物理上不可能）：**
- 开车 + 口交/性爱 → 改为「停在路边的车里」
- 摩托车行驶中 + 任何性行为 → 改为「停在路边」
- 游泳中 + 任何性行为 → 改为「泳池边/浅水区靠着边」
- 骑车/滑板等运动中 + 性行为 → 改为「停下来之后」
- 需要双手操控的活动中 + 被动享受 → 改为「活动结束后」或「被单方面服务」

**✅ 改进公式：**
- ❌ `driver driving, BJ from backseat` → ✅ `parked car backseat, man sitting, woman kneeling between legs`
- ❌ `motorcycle riding, sex from behind` → ✅ `parked motorcycle, woman bent over seat, roadside night`
- ❌ `swimming pool 69` → ✅ `poolside towel, 69 position, legs in water`
- ❌ `playing piano + blowjob` → ✅ `at piano bench, woman under piano, hands resting on keys`
